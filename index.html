NanoTable Documentation

A beautiful, lightweight, dependency-free Datatable library for modern browsers. Built for uncompromising performance, seamless server-side/client-side data handling, and a minimal footprint.

Download Minified (JS) View Live Demos

1. Installation

Because NanoTable has absolutely zero dependencies, installation is as simple as downloading the file and importing it into your project.

HTML 
<!-- Option 1: Using standard module import -->
<script type="module">
    import NanoTable from './path/to/nanotable.js';
    // Your code here...
</script>

2. Initialization

From an existing DOM Table

NanoTable can automatically extract columns and data from an existing HTML table. Simply target the table container. Ensure your headers are correctly placed within a <thead> block. Clicking the headers will automatically allow you to sort the data!

HTML / JS 
<table id="myTable">
    <thead>
        <tr>
            <th data-key="id">ID</th>
            <th data-key="name">Name</th>
            <th data-key="email">Email</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>1</td>
            <td>John Doe</td>
            <td>john@example.com</td>
        </tr>
    </tbody>
</table>

<script type="module">
    import NanoTable from './nanotable.js';
    
    const table = new NanoTable('#myTable', {
        searchable: true,
        pageSize: 10,
        theme: 'indigo' // Activate built-in themes instantly!
    });
</script>

3. Configuration Options

Option Type Default Description
dataArray[]Array of objects representing rows. Ignored if server-side is enabled.
columnsArray[]Array of configurations: { key, title, sortable, render }. Set sortable: false to disable sorting on a column.
pageSizeNumber10Number of rows displayed per page.
serverSideBooleanfalseSet to true to enable server-side processing for pagination and filtering.
ajaxMixednullURL string, config object {url, method}, or an async function handling fetching.
searchableBooleantrueEnable or disable the global search bar.
exportableBooleantrueEnable export dropdown (supports CSV, Excel, Word, PDF).
selectableBooleanfalseAdds a leading checkbox column for row selection.
responsiveBooleantrueAuto-hides overflowing columns into an expandable child row on small screens.
themeString'default'Selects the visual style. Options: 'default', 'dark', 'indigo', 'material'.

4. Complete Examples (Live)

A. Client-Side Setup (Corporate Indigo Theme)

A full setup defining NanoTable using an array of JavaScript objects. This approach does all searching, sorting, and pagination locally in the browser. Click the headers to test the sorting.

JavaScript 
const dataset = [
    { id: 101, name: 'Sarah Connor', position: 'Engineer', salary: '$90,000' },
    { id: 102, name: 'John Smith', position: 'Designer', salary: '$75,000' },
    { id: 103, name: 'Mike Johnson', position: 'Manager', salary: '$110,000' },
    { id: 104, name: 'Emily Davis', position: 'Developer', salary: '$85,000' },
    { id: 105, name: 'Robert Brown', position: 'QA Analyst', salary: '$70,000' }
];

const table = new NanoTable('#demo-client-table', {
    data: dataset,
    pageSize: 3,
    searchable: true,
    selectable: true,
    exportable: true,
    theme: 'indigo', // USING INDIGO THEME
    columns: [
        { key: 'id', title: 'ID', sortable: true },
        { key: 'name', title: 'Employee Name', sortable: true },
        { key: 'position', title: 'Role', sortable: true },
        { 
            key: 'salary', 
            title: 'Salary', 
            sortable: true,
            render: (val) => `<strong style="color: #10b981;">${val}</strong>` 
        }
    ]
});

B. Server-Side Setup (Midnight Dark Theme)

Configuring NanoTable to load data from an API endpoint. Pagination, global searching, and sorting are delegated to the server.

JavaScript 
const serverTable = new NanoTable('#demo-server-table', {
    serverSide: true,
    ajax: 'https://api.yourdomain.com/v1/users', // Method defaults to GET
    pageSize: 5,
    searchable: true,
    theme: 'dark', // USING DARK THEME
    columns: [
        { key: 'user_id', title: '#ID', sortable: true },
        { key: 'full_name', title: 'Name', sortable: true },
        { key: 'email_address', title: 'Email', sortable: true },
        { 
            key: 'account_status', 
            title: 'Status',
            sortable: false, // Disable sorting on this specific column
            render: (val) => {
                const color = val === 'Active' ? '#10b981' : '#ef4444';
                return `<span style="color: ${color}; font-weight: 600;">${val}</span>`;
            }
        }
    ]
});

5. Server-Side Data Format

When serverSide: true is configured, NanoTable sends specific payloads and expects a standardized JSON response format from your backend.

Request Payload

GET Request: Parameters are automatically appended to the query string (e.g., ?page=1&pageSize=10&search=John&sortColumn=name&sortDesc=false).

POST Request: Sent as a JSON body (application/json).

JSON Request Body (POST) 
{
  "page": 1,
  "pageSize": 10,
  "search": "john",
  "sortColumn": "name",
  "sortDesc": false
}

Expected JSON Response

JSON Response Payload 
{
  "data": [
    { "id": 1, "name": "John", "status": "active" },
    { "id": 2, "name": "Jane", "status": "inactive" }
  ],
  "total": 150
}

6. Laravel Integration

A. Laravel GET Request

UserController.php 
<?php
namespace App\Http\Controllers;

use Illuminate\Http\Request;
use App\Models\User;

class UserController extends Controller
{
    public function index(Request $request)
    {
        $page = $request->input('page', 1);
        $pageSize = $request->input('pageSize', 10);
        $search = $request->input('search');
        $sortColumn = $request->input('sortColumn');
        $sortDesc = $request->input('sortDesc') === 'true';

        $query = User::query();

        // Global Search
        if ($search) {
            $query->where(function($q) use ($search) {
                $q->where('name', 'like', "%{$search}%")
                  ->orWhere('email', 'like', "%{$search}%");
            });
        }

        // Sorting
        if ($sortColumn && $sortColumn !== 'null') {
            $query->orderBy($sortColumn, $sortDesc ? 'desc' : 'asc');
        }

        $total = $query->count();
        $data = $query->skip(($page - 1) * $pageSize)->take($pageSize)->get();

        return response()->json([
            'data' => $data,
            'total' => $total
        ]);
    }
}

7. CodeIgniter 4 Integration

A. CI4 GET Request

Users.php (Controller) 
<?php
namespace App\Controllers;

use CodeIgniter\RESTful\ResourceController;

class Users extends ResourceController
{
    public function getUsers()
    {
        $db      = \Config\Database::connect();
        $builder = $db->table('users');

        $page = $this->request->getGet('page') ?? 1;
        $pageSize = $this->request->getGet('pageSize') ?? 10;
        $search = $this->request->getGet('search');
        $sortColumn = $this->request->getGet('sortColumn');
        $sortDesc = $this->request->getGet('sortDesc') === 'true';

        // Global Search
        if (!empty($search)) {
            $builder->groupStart()
                    ->like('name', $search)
                    ->orLike('email', $search)
                    ->groupEnd();
        }

        if (!empty($sortColumn) && $sortColumn !== 'null') {
            $builder->orderBy($sortColumn, $sortDesc ? 'DESC' : 'ASC');
        }

        $total = $builder->countAllResults(false);
        $data = $builder->limit($pageSize, ($page - 1) * $pageSize)->get()->getResult();

        return $this->response->setJSON([
            'data' => $data,
            'total' => $total
        ]);
    }
}
